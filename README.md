# 🎓 Scholarship Evaluation

高校综合奖学金自动化评定系统 —— 从成绩 Excel 一键生成排名、等级分配与获奖名单。

---

## 功能概览

| 步骤 | 说明 |
|------|------|
| 1. 数据清理 | 删除成绩作废记录 |
| 2. 无资格识别 | 低分（<60）/ 旷考 / 取消考试资格 / 缓考超限（>2 门） |
| 3. 合并成绩 | 正考 + 缓考合并，剔除网络平台课程（超星/尔雅/智慧树） |
| 4. 筛选参评 | 按入学年级过滤，剔除留学生，应用各专业课程白名单 |
| 5. 按专业拆分 | 输出各专业成绩数据 Excel |
| 6. 绩点计算 | 学分加权裸绩点 = Σ学分绩点 / Σ学分 |
| 7. 裸绩排名 | 按专业分配 特等/一等/二等/三等/单项 名额 |
| 8. 综合排名 | 加载加分表 → 综合绩点 → 综合等级（含升降档约束） |
| 9. 预算验证 | 统计各等级人数 × 金额，核对总预算盈余 |
| 10. 同分异级 & 倒挂检测 | 检测综合绩点相同但等级不同、或绩点更高等级反而更低的情况 |
| 11. 自动调整（可选） | 在预算内贪心分配，自动修正同分异级和倒挂 |
| 12. 报告输出 | Markdown 日志 + 各专业排名 Excel + 获奖名单（附件5） |

## 快速开始

### 1. 创建环境

```bash
conda env create -f environment.yaml
conda activate scholarship
```

### 2. 准备数据

将以下文件放入 `data/input/`：

| 文件 | 说明 |
|------|------|
| `XX级裸绩点正考.xlsx` | 正考成绩表（必须） |
| `XX级裸绩点缓考.xlsx` | 缓考成绩表（必须） |
| `综合加分绩点.xlsx` | 加分简表（可选，二选一） |
| `综合奖学金加分表单.xlsx` | 加分表单（可选，二选一） |

> 正考/缓考文件需包含列：`成绩, 学号, 姓名, 专业, 学院, 课程名称, 学分, 学分绩点, 年级, 学期` 等。

### 3. 配置参数

编辑 `src/config.py`：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `SEMESTER_LABEL` | 学期标识，影响输出文件名前缀 | `'2025~2026第一学期'` |
| `LEVELS` / `RATIOS` | 奖学金等级及各等级占比 | 特等3%/一等5%/二等10%/三等15%/单项10% |
| `MAJOR_COURSE_WHITELIST` | 各专业计分课程白名单 | 已配置 6 个专业 |
| `BUDGET_STUDENT_COUNT` | 年级总人数（含无资格） | `485` |
| `BUDGET_PER_STUDENT` | 每人平均预算（元） | `127.5` |
| `AWARD_AMOUNTS` | 各等级奖金 | 特等1500/一等800/二等400/三等200/单项100 |
| `LOW_SCORE_THRESHOLD` | 低分无资格阈值 | `60` |
| `SPECIAL_SCORE_MIN` | 特等资格全科最低分 | `85` |
| `MAX_MAKEUP_EXAMS` | 缓考超限门数 | `2` |

### 4. 运行

```bash
cd src

# 默认（截断法取整）
python run.py

# 四舍五入
python run.py --rounding round

# 进一法
python run.py --rounding ceil

# 启用自动调整（结果保存到 data/output_adjusted/）
python run.py --auto-adjust

# 组合使用
python run.py --rounding ceil --auto-adjust
```

### 命令行参数

| 参数 | 说明 | 可选值 | 默认 |
|------|------|--------|------|
| `--rounding` | 名额取整方式 | `floor`（截断）/ `round`（四舍五入）/ `ceil`（进一法） | `floor` |
| `--auto-adjust` | 启用自动调整 | 开关，无需赋值 | 关闭 |

### 5. 输出

默认输出到 `data/output/`，启用 `--auto-adjust` 时输出到 `data/output_adjusted/`：

```
data/output/                          # 或 data/output_adjusted/
├── 排名/
│   ├── 2025~2026第一学期_校综合奖学金.xlsx              # 全专业汇总
│   ├── 2025~2026第一学期_校综合奖学金_人工智能.xlsx      # 各专业独立排名
│   └── ...
├── 成绩数据/
│   ├── 2025~2026第一学期_成绩数据_人工智能.xlsx
│   └── ...
├── logs/
│   └── run_YYYYMMDD_HHMMSS.md       # 完整运行日志（Markdown）
├── 2025~2026第一学期_无资格名单.xlsx
└── 附件5：南京工业大学本科学生综合、单项奖学金获奖名单.xlsx
```

Excel 文件自动根据等级染色（特等红、一等紫、二等蓝、三等黄、单项青、无资格深红）。

## 核心规则

### 名额计算

各专业独立计算：`名额 = fn(专业人数 × 比例)`，其中 `fn` 由 `--rounding` 决定（截断/四舍五入/进一法），每档至少 1 人。

### 裸绩等级

按 `总学分绩点`（Σ学分绩点）降序排名，依次分配特等→一等→二等→三等→单项→未获得奖学金。

- **特等**要求全科最低分 > 85

### 综合等级

1. 综合绩点 = 裸绩点 + 综合加分
2. 按综合绩点降序分配等级，但受以下约束：
   - **升档上限**：以同裸绩点中最高裸绩等级为基础，最多升一档，且不可达特等
   - **单项/未获奖升档**：只能升至三等（单项是独立类别）
   - **后 50% 限制**：裸绩排名后 50%（按专业总人数含无资格计算），综合等级上限为单项
   - **同分最佳排名**：同裸绩点组使用组内最佳排名判定 50% 分界，避免排序差异造成不公

### 同分异级检测

按专业分组，在**综合绩点**完全相同的学生中，若存在不同综合等级，则标记为同分异级。检测同样遵守升档上限约束：

- 已达到升档天花板的学生不会被建议调整
- 裸绩排名后 50% 的学生天花板为单项

### 倒挂检测

在综合排名中，若综合绩点更高的学生等级反而更低（被后面的人"超车"），则标记为倒挂。同样受升档上限和 50% 约束过滤。

### 自动调整（`--auto-adjust`）

启用后，系统在预算内按花费从低到高贪心分配：

1. 先调整同分异级（按需额外支出升序）
2. 再修正倒挂
3. 预算不足时跳过剩余项

调整后的结果保存到独立的 `data/output_adjusted/` 目录，**不影响默认输出**。

### 预算验证

- 总预算 = 年级总人数 × 每人平均预算（默认 485 × 127.5 = 61,837.5 元）  
- **特等单独核算**，不占总预算池
- 验证会在调整前后各执行一次

## 三种取整方式对比

| 取整方式 | 说明 | 名额倾向 | 盈余倾向 |
|----------|------|----------|----------|
| `floor`（截断） | `int(n × r)` | 少 → 盈余多 | 最多 |
| `round`（四舍五入） | `round(n × r)` | 适中 | 适中 |
| `ceil`（进一法） | `math.ceil(n × r)` | 多 → 盈余少 | 最少 |

> 名额越多，等级切割线越多，产生同分异级的概率越大。
